### Вопросы:
## 1. Что делает malloc?
`malloc` (memory allocation) выделяет блок памяти заданного размера в байтах и возвращает указатель на начало этого блока. Содержимое памяти не инициализируется (там лежит "мусор").

## 2. Чем отличается malloc от calloc?
| `malloc` | `calloc` |
|----------|----------|
| Выделяет один блок памяти | Выделяет память под массив из n элементов |
| Не инициализирует память | Инициализирует все байты нулями |
| Принимает 1 аргумент (размер в байтах) | Принимает 2 аргумента (количество, размер элемента) |
| `malloc(n * sizeof(int))` | `calloc(n, sizeof(int))` |

## 3. Что делает realloc?
`realloc` изменяет размер ранее выделенного блока памяти. Может:
- Увеличить блок — старая память сохраняется, новая не инициализируется
- Уменьшить блок — лишние данные теряются
- Переместить блок в другое место, если нельзя расширить текущий

## 4. Что произойдет при утечке памяти?
Программа теряет возможность освободить занятую память (нет указателя на нее). Последствия:
- Программа потребляет все больше памяти
- Система замедляется
- В конце концов программа или система может упасть из-за нехватки памяти

## 5. Зачем нужен free?
`free` освобождает динамически выделенную память, возвращая ее системе. Без `free`:
- Утечки памяти
- Программа может съесть всю доступную RAM
- Невозможно переиспользовать память для других задач

## 6. Что произойдет при двойном освобождении памяти?
Двойной вызов `free` для одного указателя вызывает **неопределенное поведение**:
- Программа может упасть (segmentation fault)
- Может испортить менеджер памяти
- В лучшем случае — ничего не произойдет
- Это критическая ошибка, приводящая к непредсказуемым результатам

## 7. Как проверить, что malloc вернул NULL?
```c
int *arr = (int*)malloc(100 * sizeof(int));
if (arr == NULL) {
    printf("Ошибка выделения памяти!\n");
    exit(1); // или return
}
```
NULL возвращается, когда система не может выделить запрошенный объем памяти.

8. В чем отличие статического и динамического массива?
|Характеристика |Статический массив|Динамический массив|
|:---:|:---:|:---:|
|Размер|Фиксированный на этапе компиляции|Задается во время выполнения|
|Выделение памяти|На стеке (или в глобальной области)|В куче (heap)|
|Изменение размера|Нельзя|Можно через realloc|
|Время жизни|До конца области видимости|Пока не вызван free|
|Освобождение|Автоматическое|Ручное через free|
|Пример|int arr[10];|int *arr = malloc(10 * sizeof(int));|
|Ограничение|Размер известен заранее|Требует ручного управления|
