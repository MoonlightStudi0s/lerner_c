Ответы на вопросы
1. Что такое файл в языке C?
В языке C файл — это последовательность байтов, хранящаяся на диске или другом носителе. С точки зрения программы, файл представляется как поток данных (stream). C не делает различий между текстовыми и бинарными файлами на уровне абстракции — разница определяется способом интерпретации данных и режимом открытия.

2. Что такое FILE*?
FILE* — это указатель на структуру типа FILE, которая определена в <stdio.h>. Эта структура содержит информацию о файловом потоке:

Позицию чтения/записи в файле

Указатель на буфер (если используется буферизация)

Флаги состояния (достигнут ли конец файла, были ли ошибки)

Режим открытия файла

Программист не работает с полями структуры напрямую — только через указатель, передавая его в функции ввода-вывода.

4. Чем отличается текстовый файл от бинарного?
Характеристика	Текстовый файл	Бинарный файл
Содержимое	Символы (читаемые человеком)	Байты (может быть нечитаемо)
Перевод строки	Может преобразовываться (CRLF ↔ LF)	Не преобразуется
Конец файла	Специальный символ (EOF) может обрабатываться	Только по размеру файла
Применение	Конфиги, логи, исходный код	Изображения, исполняемые файлы, дампы структур
Чтение	Посимвольное или построчное	Блоками байт фиксированного размера
5. Что делают функции putc и getc?
putc(ch, FILE *stream) — записывает один символ ch в указанный файловый поток. Возвращает записанный символ или EOF в случае ошибки.

getc(FILE *stream) — читает один символ из указанного файлового потока. Возвращает прочитанный символ (как int) или EOF при достижении конца файла или ошибке.

Эти функции являются макросами (более быстрыми аналогами fputc и fgetc).

6. Что делают fgets и fputs?
fputs(const char *str, FILE *stream) — записывает строку str в файл. Не добавляет автоматически символ перевода строки. Возвращает неотрицательное число при успехе или EOF при ошибке.

fgets(char *str, int n, FILE *stream) — читает не более n-1 символов из файла в буфер str. Чтение останавливается при достижении n-1, встрече символа новой строки (который тоже сохраняется) или EOF. Всегда добавляет '\0' в конец. Возвращает str при успехе или NULL при ошибке/конце файла.

7. Что делают fprintf и fscanf?
fprintf(FILE *stream, const char *format, ...) — форматированный вывод в файл. Работает как printf, но выводит не в stdout, а в указанный файловый поток.

fscanf(FILE *stream, const char *format, ...) — форматированный ввод из файла. Работает как scanf, но читает из файла, а не из stdin. Возвращает количество успешно считанных элементов или EOF при ошибке/конце файла.

Пример:

c
fprintf(f, "%s %d", name, age);
fscanf(f, "%s %d", name, &age);
8. Что делают fread и fwrite?
fwrite(const void *ptr, size_t size, size_t count, FILE *stream) — записывает бинарные данные из памяти в файл:

ptr — указатель на данные

size — размер одного элемента в байтах

count — количество элементов

Возвращает количество успешно записанных элементов

fread(void *ptr, size_t size, size_t count, FILE *stream) — читает бинарные данные из файла в память. Параметры аналогичны fwrite. Возвращает количество успешно прочитанных элементов.

Используются для работы со структурами и массивами без преобразования в текст.

9. Что делают fseek и ftell?
fseek(FILE *stream, long offset, int whence) — перемещает указатель позиции в файле:

whence = SEEK_SET — от начала файла (offset ≥ 0)

whence = SEEK_CUR — от текущей позиции

whence = SEEK_END — от конца файла (offset ≤ 0)

Возвращает 0 при успехе, ненулевое значение при ошибке

ftell(FILE *stream) — возвращает текущую позицию указателя в файле (количество байт от начала). При ошибке возвращает -1L.

Пример определения размера файла:

c
fseek(f, 0, SEEK_END);
long size = ftell(f);
10. Что делают feof и ferror?
feof(FILE *stream) — проверяет, достигнут ли конец файла. Возвращает ненулевое значение, если конец файла был достигнут при предыдущей операции чтения.

ferror(FILE *stream) — проверяет, произошла ли ошибка при последней операции с файлом. Возвращает ненулевое значение при наличии ошибки.

Важно: feof() становится истинным только после попытки чтения за концом файла. Нельзя использовать для предсказания конца файла.

11. Почему важно закрывать файл?
Закрытие файла функцией fclose() необходимо по нескольким причинам:

Сброс буфера — данные, записанные в буфер, могут физически не попасть на диск до вызова fclose() (или fflush())

Освобождение ресурсов — каждый открытый файл занимает дескриптор в ОС; количество одновременно открытых файлов ограничено

Предотвращение повреждения данных — при аварийном завершении программы несохраненные буферы могут быть потеряны

Доступ для других процессов — некоторые ОС блокируют файл для записи другими программами, пока он открыт

Утечки памяти — структура FILE выделяется динамически и должна быть освобождена

Правило: На каждое успешное открытие fopen() должно быть одно закрытие fclose().
